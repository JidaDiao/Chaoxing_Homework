### Linux基本命令操作评分标准

**总分:100分,共5题,每题20分**

**一、通用评分原则**
1.  **完全正确（20分）:** 命令、参数、路径、文件名等均与参考答案一致（或效果等同）,且无语法错误。
2.  **空白/未作答（0分）:** 未提交任何答案。
3.  **语法错误:** 根据严重程度扣分。例如,命令与参数间缺少空格是严重错误,会导致命令无法执行;而路径、文件名等出现拼写错误属于一般错误。
4.  **效果等同:** 学生答案与参考答案命令不完全一样,但能实现题目要求,可视为正确。例如 `ps aux` 与 `ps -aux`,`grep error` 与 `grep "error"`。

**二、各题详细评分细则**

**题目1:find 命令 (20分)**
- **(20分)** 答案为 `find /var/log -name "*.log"` 或 `find /var/log -name *.log`。
- **(15分)** `find` 命令、路径和 `-name` 选项均正确,但匹配模式错误,如 `"*.conf"`。
- **(10分)** `find` 命令和 `-name` 选项正确,但路径错误,如 `/var/lowwwg` 或 `/var /log`。
- **(5分)** 使用了 `find` 命令,但缺少关键选项（如 `-name`）或存在其他严重语法错误。
- **(0分)** 未使用 `find` 命令或空白。

**题目2:grep 命令 (20分)**
- **(20分)** 答案为 `grep error /var/log/messages` 或 `grep "error" /var/log/messages`。
- **(18分)** 理解正确,但匹配模式过于严格,如 `grep "^error$"`,与“包含”要求有轻微偏差。
- **(15分)** 命令和搜索词正确,但文件路径有拼写错误,如 `messageas`。
- **(10分)** 使用了 `grep` 和正确的文件路径,但缺少搜索词。
- **(5分)** 存在严重语法错误,如命令与参数间缺少空格。
- **(0分)** 未使用 `grep` 命令或空白。

**题目3:grep -v 命令 (20分)**
- **(20分)** 答案为 `grep -v error /var/log/messages` 或 `grep -v "error" /var/log/messages`。
- **(15分)** 理解了 `-v` 的用法,但在路径或搜索词中存在拼写错误或多余字符,如 `/bar/log/messages` 或 `"^error#"`。
- **(10分)** 使用了 `grep -v`,但缺少搜索词或存在影响执行的语法错误。
- **(5分)** 未使用 `-v` 选项,答案与上一题相同或类似。
- **(0分)** 未使用 `grep` 命令或空白。

**题目4:ps 命令 (20分)**
- **(20分)** 答案为 `ps -aux` 或 `ps aux`。
- **(10分)** 未使用组合选项,而是用多个命令分开作答,如 `ps -a`、`ps -u`、`ps -x`。这表明对选项组合不熟悉。
- **(5分)** 使用的 `ps` 命令只能满足题目三项要求中的一项,如 `ps -t pts/0`。
- **(0分)** 答案错误或空白。

**题目5:管道符 | (20分)**
- **(20分)** 答案为 `ps -aux | grep "sssd"` 或等效命令。
- **(15分)** 完整使用了 `命令 | grep 关键词` 的结构,但其中一部分存在拼写错误（如 `pa` 代替 `ps`）或过滤了非要求的关键词（如 `java`）。这表明理解了管道概念但不够细心。
- **(10分)** 理解管道概念,但管道符前的命令错误或不完整,导致无法实现题目要求。
- **(5分)** 答案不完整,如仅写了 `| grep sssd`,缺少输入命令。
- **(0分)** 未使用管道符或空白。
