### DHCP服务器配置实验评分标准

**总分:100分**

**基本原则:**
1.  **内容优先:** 即使截图提交位置错误,但若在其他题目中出现了本题要求的正确内容,可酌情给分。
2.  **未作答:** 任何题目无作答（无文本、无截图）均为0分。
3.  **核心概念:** 对实验的核心知识点（如服务器静态IP、配置文件语法、客户端验证）的错误将导致较大扣分。

--- 

**题目1:配置Server01的网络以及网卡配置 (17分)**
*   **（8分）IP配置正确性:** 截图需明确显示Server01被配置为**手动/静态** IP地址。如果显示为“自动(DHCP)”,此项不得分。IP地址应为`192.168.10.1`（或其他合理的静态地址）,子网掩码为`255.255.255.0`。
*   **（5分）截图完整性:** 按要求提交两张截图。一张应为网络设置界面（如nm-connection-editor的IPv4标签页）,另一张为网络详情或`ip a`命令结果,能相互印证。
*   **（4分）虚拟网络配置:** 能从截图中（如VMware虚拟机设置）看出网络模式为“仅主机模式”或功能等效的自定义网络（如VMnet1）,以确保构建隔离的实验环境。

**题目2:安装DHCP服务器 (15分)**
*   **（10分）核心命令:** 必须包含正确的包安装命令,如 `dnf install dhcp-server -y` 或 `yum install dhcp-server -y`。命令拼写错误（如`shcp-server`）将严重扣分。
*   **（5分）辅助命令:** 包含挂载、清理缓存等辅助命令（如`mount /dev/cdrom /media`, `dnf clean all`）或检查命令（如`rpm -qa | grep dhcp`）可得满分。有轻微拼写错误（如`yun`应为`yum`）酌情扣1-2分。

**题目3:配置主配置文件 (17分)**
*   **（5分）截图内容:** 必须是 `/etc/dhcp/dhcpd.conf` 文件的内容截图。
*   **（8分）全局与作用域配置:**
    *   `subnet 192.168.10.0 netmask 255.255.255.0` 声明正确。(2分)
    *   `range` 地址池范围设置合理（如`192.168.10.31 192.168.10.100`）。(2分)
    *   `option routers` (网关) 和 `option domain-name-servers` (DNS) 配置正确。(4分)
*   **（4分）主机保留配置:** 包含为Client2设置的 `host` 块,其中 `hardware ethernet` (MAC地址) 和 `fixed-address` (保留IP) 均需正确填写。

**题目4:截图Client1自动获取的IP地址 (17分)**
*   **（10分）IP地址正确性:** 截图必须来自**客户端Client1**,且其获取的IPv4地址必须在题目3配置的`range`动态地址池范围内。
*   **（7分）信息完整性:** 截图中应能清晰看到IP地址、子网掩码、默认网关和DNS服务器地址,并与服务器配置相符。提交了与题目无关的截图（如配置文件、服务器IP）不得分。

**题目5:截图Client2获取的保留地址 (17分)**
*   **（10分）IP地址正确性:** 截图必须来自**客户端Client2**,且其获取的IPv4地址必须是题目3中为其`fixed-address`指定的**保留IP**（如`192.168.10.105`）。
*   **（7分）信息完整性:** 截图中应能清晰看到IP地址,且其显示的MAC地址应与服务器配置文件中为Client2绑定的MAC地址一致。

**题目6:截图租用信息文件 (17分)**
*   **（5分）截图内容:** 必须是 `cat /var/lib/dhcpd/dhcpd.leases` 命令的输出结果。
*   **（12分）租约信息完整性:**
    *   文件中应包含Client1（动态获取）的租约记录。(6分)
    *   文件中应包含Client2（地址保留）的租约记录。(6分)
    *   缺少任一客户端的租约信息都会导致扣分。
